# Phase C workflow invariants: gene_extractor (Wave 1 pilot, FULL coverage).
#
# Invariants are STRONGER than plain phrase retention: each line is an ordered
# step of the extractor's load-bearing builder workflow. The invariant framework
# (1) reuses the retention assertion -- every line must appear in the assembled
# render -- and (2) asserts the lines appear in this same order by first
# occurrence, so the rewrite cannot scramble the
# search -> shortlist -> record_evidence -> stage -> review/finalize -> stop
# sequence even if it keeps the individual phrases.
#
# AUTHORING RULE (enforced by the harness): each phrase must occur EXACTLY ONCE
# in the assembled render. These are verbatim from the rewritten <workflow>
# ordered block (steps 1-5) plus the closing stop-searching rule, where each
# occurs exactly once and in workflow order. (GE-22/GE-23/GE-24/GE-47/GE-25/
# GE-26/GE-54 in docs/design/phaseC-checklists/gene_extractor.md.)

# Step 1 -- retrieve candidate mentions and context first.
Search/read the document for candidate mentions and context.

# Step 2 -- shortlist focal genes before spending evidence calls.
build a short provisional list of focal genes before selecting evidence

# Step 3 -- read the chunk, pick backend span ids, record one evidence unit.
select the `evidence_spans[].span_id` values that directly support one evidence unit

# Step 4 -- stage each retained focal gene with its verified evidence ids.
Stage each retained focal gene with `stage_gene_mention_evidence`, passing the verified `evidence_record_ids`

# Step 5 -- review staged candidates before finalizing.
Call `list_staged_gene_mention_evidence` to review

# Stop rule -- once each focal gene has a verified quote, finalize.
proceed to finalization instead of searching for more support.
