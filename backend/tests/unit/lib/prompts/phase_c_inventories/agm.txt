# AGMV-12 AGM helper methods: which-method-when judgment (contract tokens)
When `subject_identifier` is present, call `map_entity_curies_to_info` first, supplying `entity_type: "agm"` and `entity_curies` with the identifier.
When `subject_label` and taxon context are present, call `map_entity_names_to_curies`, supplying `entity_type: "agm"`, `entity_names`, and `taxon_id`.

# AGMV-13..AGMV-19 bounded ordered lookup path (steps pinned in agm.invariants.txt)
If `subject_identifier` is present, call `map_entity_curies_to_info` for that identifier.
If the identifier lookup returns one AGM whose taxon does not conflict with the supplied taxon, resolve it.
If no identifier match is found and both `subject_label` and `taxon` are present, call `map_entity_names_to_curies` with `entity_type: "agm"` and `taxon_id`.
Treat a single label match as resolved only when the returned identifier, label, and taxon evidence do not conflict with the supplied fields.
If multiple candidates remain, return `status: "unresolved"` and preserve candidates.
If taxon is missing for label-only lookup, return unresolved with an explanation that AGM label routing requires taxon context.
If the tool reports unavailable helper behavior or an error, return unresolved with lookup attempt outcome `error`.

# AGMV-20 shared result contract root fields (contract tokens, incl agm-specific roots)
Populate these root fields exactly; do not wrap them under another object:
`request_id`: copy from the validator request.
`validator_binding_id`: copy from the validator request.
`validator_agent`: copy the package and agent identity from the request.
`target`: copy the target object from the request.
`resolved_values`: scalar outputs keyed by the binding's `expected_result_fields`
`resolved_objects`: database-returned AGM facts for resolved matches.
`missing_expected_fields`: the expected keys you could not fill from lookup evidence.
`candidates`: generic candidate records for exact, ambiguous, or alternate matches
`lookup_attempts`: one record for each `agr_curation_query` call
`curator_message`: a concise curator-facing summary of what resolved, what is missing, or why ambiguity remains.
`explanation`: a plain-language decision explanation tied to the database evidence and the lookup attempts.
`agm_candidates`: AGM-specific candidate detail preserving useful lookup fields
`unresolved_explanations`: the specific reasons the AGM could not be resolved.
