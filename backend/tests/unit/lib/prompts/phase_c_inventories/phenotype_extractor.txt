# PE-05 retrieval before answering

# PE-06 individual phenotype assertions supported in this paper

# PE-09 stage once per retained phenotype assertion (passage of several = one call per assertion)
Each retained phenotype is staged once with `stage_phenotype_observation`, after `record_evidence(span_ids=[...])` creates its supporting source text.

# PE-08 retained phenotype carries statement + mentions + evidence ids + pending hints
Each retained phenotype carries the free-text statement, one or more `source_mentions`, the verified `evidence_record_ids`, and -- when the paper supports them -- pending subject and phenotype-term hints

# PE-13 uncertain-identity phenotype still staged with selectors left for the validator

# PE-10 finalize exactly once

# PE-14 no invention

# PE-33 experimental-support gate
A phenotype is "experimentally supported" when this paper presents observational or quantitative data for that phenotype, including summary or conclusion statements about experiments performed in this paper.

# PE-34 include list (direct observed; quantitative/qualitative; negative; figure/table/Results)
Include a direct observed phenotype caused by a specific genotype, allele, perturbation, treatment, or condition.

# PE-35 exclude prior-work/predicted/hypothesized/narrative (cross-cutting non-gene-evidence token, VERBATIM)

Phenotypes mentioned only as previously reported, predicted, hypothesized, narrative, or review-style claims.

# PE-36 author-year citations signal prior work
Parenthetical author-year citations (e.g., "(Smith et al., 2018)") within a sentence are a strong signal the finding originates from prior work, not this paper.

# PE-37 methods-only mentions are not findings by themselves
Methods/protocol mentions (strain lists, buffer recipes, construct generation) are not findings by themselves; retain a phenotype from a methods section only when the paper also presents experimental results involving that entity.

# PE-38 wild-type/control baseline excluded
Wild-type/control baseline descriptions that do not state a mutant or perturbed phenotype assertion are excluded.

# PE-40 record evidence only for curatable phenotype findings in this paper
Call `record_evidence` only for evidence spans that support a curatable phenotype finding in THIS paper. Record evidence only for phenotypes you intend to stage.

# PE-41 strong phenotype evidence content
Describes an observed phenotype caused by a specific genotype, allele, perturbation, or treatment.

# PE-42 strong quote examples header (cross-cutting token) + examples
Strong quote examples:
"daf-2(e1370) adults produced 40% fewer progeny than wild type."

# PE-39 exclude by not staging (no metadata.* write channel); discard takes a free-text reason

# PE-43 phenotype vs disease
When a model organism phenotype is described as resembling a human disease, capture the observable trait as a phenotype and treat the disease name as context only.

# PE-44 phenotype vs normal biology
"Neurons fail to migrate to the cortex in Reelin mutants" -> phenotype (migration failure).
The key is whether the observation describes a deviation from normal caused by a specific genotype or condition.

# PE-45 polarity and severity
Distinguish "loss of X" from "gain of X" from "reduced X" from "enhanced X".
Penetrance: "fully penetrant lethality" vs "30% penetrant lethality".

# PE-46 negative results captured with negated=true
"No phenotype was observed in X mutants" IS a finding; capture it with `negated=true`.
Absence of discussion about a phenotype is NOT a negative result.

# PE-47 genotype-phenotype linking staged as the subject hint
Always associate each phenotype with the specific genotype/condition that causes it, staged as the paper-backed subject hint.
When multiple genotypes are discussed, link each phenotype to the correct genotype.

# PE-52 ground ZECO class/id CURIEs with the domain-field tools (do not guess)

`condition_chemical_curie`: the ChEBI CURIE for the chemical when a chemical treatment is stated (e.g. `CHEBI:6909`).
