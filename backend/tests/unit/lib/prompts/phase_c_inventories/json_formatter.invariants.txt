# Phase C workflow invariants: json_formatter ordered projection-tool path.
#
# Each phrase below must occur exactly once in the assembled prompt and in this
# order. These guard against drifting back to the removed raw JSON-writer path.

Call `explain_formatter_capabilities` or `inspect_output_artifacts` when you need to understand available row sources, field refs, default columns, JSON shapes, source refs, or limitations.
Use `inspect_output_rows` and `inspect_field_values` as needed to choose source-backed fields, filters, sort order, row strategy, source restrictions, and shape.
Use `build_default_projection_plan` for a safe starting point. When runtime context names a latest `extraction-result:<uuid>` and the curator did not ask for all results, pass that ref as `source_ref`; export multiple sources only when explicitly requested. Then adjust the plan only with source-backed `field_ref`, transform, filter, sort, row-source, row-strategy, JSON-shape, and filename intent.
Call `validate_output_projection`; call `preview_output_projection` when the request involves custom fields, grouping, filters, ordering, or source restrictions.
Call `finalize_and_save` exactly once after validation succeeds. Pass `plan_json` for the validated plan and `filename_hint` when a clear filename is available.
If required rows, fields, or structure are missing, ambiguous, or unsupported, call `formatter_cannot_complete` with a precise reason and do not save a file.
