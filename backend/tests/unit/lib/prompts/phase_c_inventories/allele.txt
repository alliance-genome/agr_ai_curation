# AV-15 CRITICAL never-assume-equivalence + literal-symbol-first
NEVER assume a symbol is an abbreviation for another gene name. ALWAYS search the LITERAL symbol from the paper first.
FIRST search the exact compact symbol, notation, or ID from the paper.
ONLY IF that fails with zero results, consider alternative interpretations.
NEVER substitute a gene name with what you think it "really means".
The database stores these as synonyms, but only finds them when you search the literal symbol.

# AV-16 no memory / no guessing
Do not answer from memory or training data, guess allele IDs, guess symbols or names, or report allele information you did not get from a query.

do not treat the extractor's lookup hints as the final identity

# AV-20 which-method-when judgment
A CURIE or ID (for example `MGI:3689906`) -> `get_allele_by_id`.
it handles compact notations, partial matches, synonyms, and alternative names in one call
An exact full database symbol you are already confident in (such as `Ulk1<sup>tm1Thsn</sup>`) -> `get_allele_by_exact_symbol`.

# AV-21 bounded ordered lookup path
Decide whether the input is a CURIE/ID or a compact symbol/notation.
For symbol/notation input, call `search_alleles`, and pass the species `data_provider` whenever the species is known (see Species filtering).
If too many results come back, narrow by adding more characters; if you have not already scoped by the known species, add the `data_provider` now.
report "Allele not found in database" with the exact search you attempted.

# AV-22 troubleshooting judgment
A deprecated or synonym-stored symbol -> use `search_alleles`, since synonym matching may surface the current allele.
Several alleles sharing a symbol -> check the species, associated gene, and notation on each query result before deciding.
