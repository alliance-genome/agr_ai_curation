# Phase C workflow invariants: disease validator (Wave 3 — bounded lookup path).
#
# Invariants are STRONGER than plain phrase retention: each line is an ordered
# step of the validator's bounded Disease Ontology lookup path. The invariant
# framework (1) reuses the retention assertion -- every line must appear in the
# assembled render -- and (2) asserts the lines appear in this same order by first
# occurrence, so the rewrite cannot scramble the
# DOID-lookup -> name-search -> too-many-trim -> no-match -> tool-error sequence.
#
# AUTHORING RULE (enforced by the harness): each phrase must occur EXACTLY ONCE in
# the assembled render and be listed in workflow order. These are verbatim from the
# rewritten <lookup_workflow> "Bounded lookup path" numbered block (steps 1-5),
# where each occurs exactly once and in order. (DISV-17a..DISV-17e in
# docs/design/phaseC-checklists/disease.md.)

# Step 1 -- DOID input: exact CURIE lookup with DOTerm type.
For a DOID input, call `get_ontology_term` with `term: <DOID>` and `ontology_term_type: "DOTerm"`.

# Step 2 -- disease name input: typed label/synonym search.
For a disease name input, call `search_ontology_terms` with `term: <label>`, `ontology_term_type: "DOTerm"`, `include_synonyms: true`, and a small `limit`.

# Step 3 -- too many matches: trim to top 10 alphabetically, preserve as candidates.
If many matches remain, return the top 10 alphabetically by name and preserve them as candidates.

# Step 4 -- no tool-backed match for an expected field: unresolved, list it as missing.
If no tool-backed match is found for a requested expected field, return `status: "unresolved"` and list that field in `missing_expected_fields`.

# Step 5 -- tool lookup fails after one retry: unresolved with an error attempt.
If the package-tool lookup fails after one retry, return `status: "unresolved"` with an `error` lookup attempt and a curator-safe message.
